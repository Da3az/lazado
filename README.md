# lazado

Interactive Azure DevOps CLI powered by [fzf](https://github.com/junegunn/fzf). Manage work items, pull requests, pipelines, and repos from your terminal.

![lazado](https://img.shields.io/badge/shell-bash-green) ![license](https://img.shields.io/badge/license-MIT-blue)

## Features

- **Interactive navigator** — type `ado` to launch an fzf-powered menu
- **Work items & user stories** — list, create, search, update status with dynamic state discovery
- **Pull requests** — list, create, approve, merge, checkout source branch
- **Auto-linking** — PRs auto-link to work items based on branch naming (e.g., `feature/1234-fix-bug`)
- **Smart detection** — auto-detects organization and project from your git remote
- **Config-driven** — global config, per-project config, environment variable overrides
- **Cross-platform** — Linux and macOS

## Prerequisites

- [Azure CLI](https://learn.microsoft.com/en-us/cli/azure/install-azure-cli) with the [DevOps extension](https://learn.microsoft.com/en-us/azure/devops/cli/)
- [jq](https://jqlang.github.io/jq/)
- [fzf](https://github.com/junegunn/fzf)
- [git](https://git-scm.com/)

```bash
# Install prerequisites (Ubuntu/Debian)
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash
az extension add --name azure-devops
az login
sudo apt install jq
# fzf: https://github.com/junegunn/fzf#installation
```

## Installation

### Using curl (recommended)

```bash
curl -o- https://raw.githubusercontent.com/da3az/lazado/main/install.sh | bash
```

### Using git

```bash
git clone https://github.com/da3az/lazado.git ~/.lazado
```

Add to your `~/.bashrc`:

```bash
export LAZADO_DIR="$HOME/.lazado"
[ -s "$LAZADO_DIR/lazado.sh" ] && \. "$LAZADO_DIR/lazado.sh"
```

### Manual

```bash
mkdir -p ~/.lazado
curl -L https://github.com/da3az/lazado/archive/main.tar.gz | tar -xz --strip-components=1 -C ~/.lazado
```

Then add the same lines to `~/.bashrc` as above.

### First-time setup

After installation, restart your terminal and run:

```bash
ado-init
```

The setup wizard will:
1. Auto-detect your organization and project from your git remote
2. Prompt for configuration values
3. Validate the connection
4. Cache work item states
5. Write your config to `~/.config/lazado/config`

## Usage

### Interactive navigator

```bash
ado
```

Launches an fzf menu with all available actions:

```
My Work Items
My User Stories
Create Work Item
Create User Story
Update Item Status
Search Items
Current Branch Ticket
─────────────────────
Pull Requests (this repo)
My Pull Requests (all repos)
Create Pull Request
─────────────────────
Pipelines
Repos
Open in Browser
```

### Direct commands

#### Work items

```bash
ado-wi 4089               # View work item
ado-wi-detail 4089         # View full detail (JSON)
ado-wi-branch              # View work item linked to current branch
ado-wi-search "login bug"  # Search by title
ado-wi-state 4089 Active   # Update status
```

#### Pull requests

```bash
ado-prs                    # List open PRs for current repo
ado-my-prs                 # List my PRs across all repos
ado-pr 123                 # Show PR details
ado-pr-create              # Create PR from current branch (prompts for title)
ado-pr-create develop      # Create PR targeting 'develop'
ado-pr-approve 123         # Approve a PR
ado-pr-complete 123        # Complete/merge a PR
ado-pr-web 123             # Open PR in browser
```

#### Other

```bash
ado-pipelines              # List recent pipeline runs
ado-repos                  # List all repos
ado-web                    # Open current repo in browser
ado-refresh-states         # Refresh cached work item states
ado-help                   # Show command reference
```

## Configuration

### Config file

Generated by `ado-init` at `~/.config/lazado/config`:

```bash
org=https://dev.azure.com/MyCompany
project=MyProject
clone_dir=$HOME/dev
branch_prefix=feature
default_target_branch=main
cache_ttl=86400
```

### Project-level config

Create `.adoconfig` at your repo root to share settings with your team:

```bash
project=MyProject
default_target_branch=main
branch_prefix=feature
```

### Environment variables

All config values can be overridden with `LAZADO_` prefixed environment variables:

| Variable | Description | Default |
|---|---|---|
| `LAZADO_ORG` | Organization URL | (auto-detected from git remote) |
| `LAZADO_PROJECT` | Project name | (auto-detected from git remote) |
| `LAZADO_CLONE_DIR` | Directory for cloning repos | `$HOME/dev` |
| `LAZADO_BRANCH_PREFIX` | Prefix for new branches | `feature` |
| `LAZADO_DEFAULT_TARGET_BRANCH` | Default PR target branch | `main` |
| `LAZADO_CACHE_TTL` | State cache TTL (seconds) | `86400` |
| `LAZADO_CACHE_DIR` | Cache directory | `$HOME/.cache/lazado` |
| `LAZADO_BROWSER_CMD` | Browser open command | `xdg-open` / `open` |
| `LAZADO_FZF_OPTS` | Extra fzf options | (empty) |
| `LAZADO_SSH_URL_TEMPLATE` | SSH clone URL template | `git@ssh.dev.azure.com:v3/{org}/{project}/{repo}` |

### Config loading order

1. Built-in defaults
2. Global config (`~/.config/lazado/config`)
3. Project config (`.adoconfig` in repo root)
4. Environment variables
5. Auto-detection from git remote URL

Higher priority sources override lower ones.
